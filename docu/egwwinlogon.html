Kurzübersicht

Die Anwendung EGroupware wird von einem zentralen Webserver nach außen präsentiert. Als „Client” des Anwenders dient ein Standard Webbrowser. Damit ist die Anwendung EGroupware universell verfügbar.

Als Zusatz zur Verwaltung von Windows-Benutzern kann im Egroupware ein weiteres Modul installiert werden. Auf dem Betriebssystem wird PGina und ein weiteres Plugin installiert.

Vorteile:

  • Einfaches Protokoll (HTTP(s)+JSON)
  • Offline Benutzerliste liegt nach ersten Login auf den System vor ...mehr (TODO)
  • Erweiterbar mit Systembefehlen & Einstellungen
  • Loging kontrolle (Hardware & Rechte)
  • Erweiterbar für dritte Entwickler (Scripte)

Installation pGina

Führen Sie das normale Setup von pGina aus.

Installieren Sie Java SDK 1.8.*.

Windows 7/Vista

Nach der Installation gehen Sie in das pGina Verzeichnis.

Kopieren Sie anschließend aus dem "pGina EGroupware Plugin"-Packet die DLL ins den: 

    <pgina>/Plugins/Core/

Achtung, prüfen Sie vorher die Eigenschaft der DLL, ob Windows diese blockiert. Klicken Sie, wenn ja, auf "Zulassen".

Kopieren Sie die folgenden Dateien/Ordner ins pGina Hauptverzeichnis. 

    
    
        
  • lib
    • 
        
  • egwwinlogon.jar
    • 
        
  • jni4net*
    • 
        
  • log4*
    •

Nach dem Kopieren der Daten, starten Sie pGina Konfiguration. Das Plugin sollte nun in der Liste aufgeführt werden.

Ist das Plugin vorhanden stellen Sie den Aufgabenbereich ein. Sollte nur Egroupware zum Login benutzt werden, dann nehmen Sie die Einstellungen wie im folgenden Bild vor.
Achtung, das Egroupware Plugin sollte nicht auf "Gateway" reagieren, da die Aktuelle Version dies nicht umsetzten kann. Stattdessen übernihmt "Local Machine" diese Aufgabe vorläufig.

Konfigurieren Sie nun das Plugin "Egroupware".
Achtung 1, die Egroupware URL muss mit einen "/" enden.
Achtung 2, bei HTTPS ist es wichtig das die Domain in der URL mit der Zertifikat Domain übereinstimmt, bsp. mit www "www.meinedomain.de"
Achtung 3, "Start Java App by Login" startet eine Java Application + Browser mit Egroupware, ist dieses nicht gewünscht, so muss die Box deaktiviert werden.

Als letztes Starten Sie den "pGina Service" im Windows neu.

Installation pGina Egroupware

Der Modul-Ordner wird ins EGroupware Verzeichnis abgelegt. Über Setup wird das Modul anschließend Installiert.

Im EGroupware sollte jeden Benutzer die Rechte auf das Modul bekommen.
Achtung, wenn ein Benutzer keine Rechte auf das Modul hat, kann dieser Benutzer die Offline Benutzerliste vom EGroupware nicht im Betriebssystem benutzen.

pGina Einstellungen

Passen Sie das Logoimage an, in den Packeten von HW, liefern wir das Egroupware Logo mit.
Wenn Sie Ihr eigenes Logo (von Ihrer Firma) verwenden wollen, muss dieses in "Bmp"-Format vorliegen.
Der optimalste Ordner für das Logoimage ist der Hauptordner von PGina.

Aktualisierung Ihrer Installation

Vor dem Updaten sollte der "pGina Service" angehalten werden. Der pGina Konfigurator sollte geschlossen sein.

Ein Update der Plugin DLL kann erst vorgenommen werden, wenn keine Instance dieser läuft.

Gehen Sie dann wie bei der Installation vor.

Sicherheitshinweiße

Wichtig ist das Egroupware über das Protokoll "https" abgerufen wird!
